大家好，我是村长。

## **启动界面**

打开 Visual Studio，先看启动界面。

这里其实只干两件事：

👉 创建新项目
👉 打开已有项目

我们今天，选择第一种。

## **创建新项目**

点击：👉“创建新项目”

这里有很多模板，这些模板其实就是官方预定义了“程序结构“的文件，帮助我们快速创建应用程序。

右上角筛选一下：

* 语言：C#
* 平台：Windows
* 类型：控制台

控制台程序没有界面，非常适合新手练语法。

这里会看到两个“控制台应用”：

上面模板支持跨平台，下面支持Windows平台——我们选下面这个，点击下一步” 。


## 配置新项目

进入“配置新项目”窗口，这里配置三个东西：解决方案名称、项目名称和位置

1. 解决方案名称 → 改成 LearnCSharp

解决方案是“书包”，里面可以装多本书（项目）。
2. 项目名称 → 改成 Lesson1

项目是“书”，里面装的是真正的代码。
3. 位置 → 点“三点”图标，选桌面，方便找到。

💡 小技巧：勾选“将解决方案和项目放在同一目录中”，可以减少文件夹层级，结构更清爽。
命名建议：用帕斯卡命名法，每个单词首字母大写，比如 LearnCSharp、Lesson1。
4. 框架 → 保持默认 .NET Framework 4.8.1 就行。

确认无误后，点击右下角 “创建”。

稍等 1-2 秒，右侧“解决方案资源管理器”里会自动生成项目文件——项目结构就搭好了。



首先把解决方案的名称修改为：LearnCSharp，解决方案是容器，里面可以存放多个项目，解决方案就像书包，项目就像一本书，一个书包可以装很多本书。单本书时，可以不需要书包，勾选‘将解决方案和项目放在同一目录中’这样可以减少文件夹层级，结构更简洁。

接下来，再把项目名称修改为“Lesson1",项目是存放代码的容器，一个项目就是一个功能模块，就像一本书。多个项目就组成了一个应用程序。

位置: 是存储解决方案的本地路径。在这里，我希望把项目存储在桌面上：点击后面的“三点”图标，选择桌面，点击“选择文件夹”，这样就设置好了解决方案的存储位置。

注意，无论是解决方案还是项目，建议使用帕斯卡命名法命名，即名称中的每个单词的首字母大写。

- 框架：这里保持默认的.NET Framework 4.8.1，这是最稳定的Windows兼容版本。

最后，确认选项无误后，点击右下角的‘创建’按钮。稍等 1-2 秒，Visual Studio 会自动生成项目文件，可以在右侧解决方案资源管理器，看到自动生成文件”——这样，我们就成功的创建好了项目的结构。”

## 运行第一个项目

`Program.cs` 文件是整个程序主文件，Program文件中的Main方法是程序启动时的入口”。

记住哦，程序总是从这个Main方法开始执行的！这是程序执行的起点！

找到Main方法，在Main方法的花括号内输入"Console.WriteLine("Hello World");

输入完成后，点击上方工具栏中的“开始执行(不调试)”按钮，或者按下快捷键Ctrl + F5，运行程序。

如果在弹出的命令行窗口中看到“Hello World”，恭喜你！说明我们的项目构建成功。


## 添加第二个项目

现在我们已经能创建单个项目了，但实际开发中常需要多个项目模块协作”

和创建第一个项目步骤完全一样，

- 我们在“解决方案”上单击右键

- 找到“添加”

- 选择“新建项目”

- 选择适合windows平台的“控制台应用”，单击下一步

- 配置新项目
      - 项目名称填写：Lesson1B
      - 位置和框架: 保持默认
      - 单击创建
- 在Lesson3的Program主文件中，再次找到Main方法，在Main方法的花括号内输入"Console.WriteLine("Hello C#!");

这样就完成了第二个项目的添加

## 设为启动项目

点击顶部工具栏中的“开始执行”按钮后，你会发现，当前执行的项目并不是Lesson1B，而是Lesson1。

这是因为：Lesson1被设Visual Studio设置为了默认启动项目，默认启动项目的名称会加粗显示。

一个解决方案里有多个项目时，每次只能有一个是启动项目

如果你想运行的是 Lesson1B，就需要手动把它设置为‘启动项目，你可以：

- 右键单击Lesso3项目
- 选择“设为启动项目”
- 现在，点击“开始执行”按钮，Lesson3项目就可以启动了。
