## **开场**

大家好，我是村长。

👉 这一节，我们正式写下**人生第一行C#代码**。

## **启动界面**
```
打开 Visual Studio，先看启动界面。

在启动界面下，其实只干两件事：

👉 创建新项目
👉 打开已有项目

我们今天，选择第一种。
```

## **创建新项目**
```
点击：“创建新项目”

这里有很多模板，这些模板其实就是官方预定义的“程序启动”文件，帮助我们快速创建应用程序。

右上角筛选一下：

* 语言：C#
* 平台：Windows
* 类型：控制台

由于控制台程序没有界面，非常适合新手练语法。

所以我们选择“创建控制台应用”

这里会看到两个“控制台应用”：

上面模板支持跨平台，下面只支持Windows平台——我们选下面这个，点击下一步” 。
```

## 配置新项目
```
进入“配置新项目”窗口，这里配置三个东西：解决方案名称、项目名称和位置

1.首先把解决方案的名称修改为：LearnCSharp，解决方案名称就是你的应用程序的名称，解决方案是个容器，可以包含多个子项目。就像一本书可以包含多个章节。解决方案如果一本书只有一个章节，可以不需要书名，勾选‘将解决方案和项目放在同一目录中’这样可以减少文件夹层级，结构更简洁。

2. 接下来，再把项目名称修改为“Lesson1",项目是存放代码文件的容器，一个项目就是一个功能模块，就像一本书的某个章节，多个项目就组成了一个应用程序。

项目是“书”，里面装的是真正的代码。

3. 位置: 是存储解决方案的本地路径。在这里，我希望把项目存储在桌面上：点击后面的“三点”图标，选择桌面，点击“选择文件夹”，这样就设置好了解决方案的存储位置。。

4. 框架：这里保持默认的.NET Framework 4.8.1，这是最稳定的Windows兼容版本。

最后，确认无误后，点击右下角的‘创建’。稍等 1-2 秒，

右侧“解决方案资源管理器”里会自动生成项目文件——这样项目结构就搭好了。


💡 小技巧：无论是解决方案还是项目，建议使用帕斯卡命名法命名，即名称中的每个单词的首字母大写。
```


## 运行第一个项目

`Program.cs` 文件是整个程序主文件，Program文件中的Main方法是程序启动时的入口”。

记住哦，程序总是从这个Main方法开始执行的！这是程序执行的起点！

找到Main方法，在Main方法的花括号内输入"Console.WriteLine("Hello World");

输入完成后，点击上方工具栏中的“开始执行(不调试)”按钮，或者按下快捷键Ctrl + F5，运行程序。

如果在弹出的命令行窗口中看到“Hello World”，恭喜你！说明我们的项目构建成功。
