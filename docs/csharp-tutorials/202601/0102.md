## **创建项目**
```
双击打开 Visual Studio，首先看到的是启动界面。

在启动界面下，通常只做两件事：

要么创建一个新项目 要么打开电脑中已有的旧项目

点击：“创建新项目” 

你会看到一个模板列表，“模板“就是一组帮助我们快速创建项目结构的文件。

上方是模板筛选器：

* 语言：C#
* 平台：Windows
* 项目类型：控制台

控制台是一个黑底白字的命令行窗口，没有复杂的图形界面，非常适合初学者学习 C#语法。

“接下来你会看到两个‘控制台应用’模板。请务必选择上面那个，它支持跨平台开发，并且基于现代的 .NET 框架。而下面这个只支持 Windows 平台，用的是传统的 .NET Framework。

选好后，点击‘下一步’。”
```

## 配置新项目

```
在配置新项目窗口下

首先把解决方案的名称改为：L-earn-C-Sharp，也就是 LearnCSharp

然后把项目名称修改为“Lesson1"，

注意，关于解决方案和项目的名字，微软官方推荐使用帕斯卡命名风格，也就是名称中每个单词的首字母要大写。

解决方案是个文件夹，相当于一个容器，用于组织一个或多个项目。

项目也是个文件夹，用于存放我们编写代码的文件。

你可以把解决方案想象成一个书包，里面可以装多本书，每本书就是一个项目，比如 Lesson1、Lesson2 等等。



接下来设置存储位置，我把项目存储在桌面上：点击后面的“三点”图标，选择桌面，点击“选择文件夹”，这样就设置好了存储位置。

可以在底部看到项目的具体存储路径。



如果解决方案下只有一个项目，可以勾选“将解决方案和项目放在同一目录中“ ，这样，就不需要书包了，减少了一层嵌套，结构更简洁。

确认无误后，点击‘下一步’，



框架保持默认的10.0版本，它是目前最新、最主流的框架。


注意：一定要勾选“不使用顶级语句“。顶级语句是新语法，隐藏了许多重要的概念，不利于初学者学习，所以一定要勾选上。

最后点击创建，项目开始创建，

可以看到桌面上已经生成了解决方案文件夹。

创建完成后，

如果出现Copilot 窗口，可以直接关掉。看到解决方案资源管理器，就说明项目结构搭建好了。

点击“自动隐藏“图标，可以隐藏“资源管理器“。再点一下标签，可以显示出来

如果不小心关闭了资源管理器，

可以通过窗口菜单下的“重置窗口布局“找回来。

```

## 创建第一个项目

创建完解决方案后，默认打开的是 `Program.cs` 文件，

为了方便观看，我们调整一下左下角的显示比例，改为 150%。

`Program`文件是整个程序的主文件，因为它包含了启动程序的结构代码：

注意看：这里有三组花括号，最内层的花括号是属于Main()方法。

Main方法是个特殊的方法，特殊在于：它是程序的入口，也就说：程序每次执行，都从Main 方法开始！

这意味着：我们写的代码必须要放在 Main 方法的花括号里面

这行代码是模板提供的测试语句，意思是：向控制台输出一句Hello World。这样做的目的是：通过运行 Hello World 帮助我们测试代码环境是否工作正常。

什么都不用改，直接点击上方工具栏中的“开始执行“按钮（快捷键：Ctrl + F5)

如果你在弹出的黑窗口中看到“Hello World”，恭喜你，你已经成功运行了第一个 C# 程序。

按下任意键关闭窗口。

现在你不用完全理解每行代码的含义，你只需记住三件事情：

1. Program.cs 文件是项目的主文件
2. Main()方法是程序执行的入口
3. 运行程序的快捷键是：Ctrl + F5 

点叉关闭Program文件，在资源管理器中，双击 Program 文件可以再次打开

## 创建第二个项目
接下来，创建第二个项目

在“解决方案资源管理器“上点右键，找到“添加“

选择“新建项目“

选择“第一个模板“，点击下一步，把项目名称改为“Lesson1a",点击下一步，框架保持默认，点击“创建“。

第二个项目就创建好了。

接下来，把第 7 行代码修改为 Hello C#，记住：先点保存，再点开始运行，养成好习惯。

这时，你会发现控制台显示的不是 Hello C#,而是第一个项目中的 Hello World。

为什么？

这是因为解决方案资源管理器，默认把第一个项目“Lesson1“设为了启动项目。

有没有发现？“Lesson1“是粗体的，而「Lesson1a」不是粗体。

粗体表示默认启动项目。

如果你想运行第二个项目，可以在Lesson1a 这个项目名上点右键，选择“设为启动项“

或者在工具栏中，直接切换到 Lesson1a也可以。

设置好之后，再次点击“运行“，Hello C#就成功输出了。


## .slnx 文件

学会创建项目后，接下来，我们点击右上角的关闭按钮，退出Visual Studio，回到桌面

打开解决方案文件夹，可以看到

两个项目文件夹和一个.slnx 文件

每个文件夹下都有一个 Program.cs 文件和一个 .csproj 文件。

.csproj 文件是项目配置文件，记录了项目信息。

.slnx 文件是解决方案配置文件，记录了解决方案的相关信息。

下次你想继续写代码，双击它可以打开整个解决方案。
