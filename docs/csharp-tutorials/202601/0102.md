## 创建项目
```
双击打开 Visual Studio，首先看到的是启动界面。

在启动界面下，通常只做两件事：

要么创建一个新项目 要么打开电脑中已有的旧项目

今天我们选第一种，点击：“创建新项目” 

这里是vs 提供的模板列表，大家记住：“模板” 就是一组提前写好的文件，能帮我们快速搭项目结构，不用从零开始写。

上方是筛选器：

* 语言：C#
* 平台：Windows
* 项目类型：控制台

控制台就是一个黑底白字的命令行窗口，没有复杂的图形界面，非常适合初学者学习语法。

“接下来你会看到两个‘控制台应用’模板。

重点来了！

一定要选上面这个——它支持跨平台开发，基于现代的 .NET 框架。

下面这个只支持 Windows，用的是传统 .NET Framework，新手不推荐。

选好之后，点击‘下一步’。”

```

## 配置项目

```
在配置新项目窗口，我们需要设置三个东西：

项目名称
存储位置
解决方案名称

首先，把解决方案名称改为：L-earn-C-Sharp（也就是 LearnCSharp）

然后把项目名称改为 Lesson1。

小贴士：解决方案和项目的名字，推荐用帕斯卡命名——也就是每个单词首字母大写。

接下来设置存储位置。我选桌面：点击后面的“三点”图标，选择桌面，再点“选择文件夹”。

底部可以看到项目的具体存储路径。

如果只有一个项目，可以勾选“将解决方案和项目放在同一目录中”，这样能减少一层嵌套，结构更简洁。

确认无误后，点击“下一步”。

框架保持默认的 10.0 版本，这是目前最新、最主流的。

关键操作来了：一定要勾选“不使用顶级语句”。

为什么？因为顶级语句是新语法，会隐藏很多重要的基础概念，不利于初学者学习。所以，务必勾选。

最后点击“创建”，项目就开始生成了。可以看一眼桌面，解决方案文件夹已经出现了。

创建完成后，如果弹出 Copilot 窗口，直接关掉。看到右边的“解决方案资源管理器”，就说明项目结构搭建好了。

补充两个小技巧：

点击“自动隐藏”图标，可以隐藏资源管理器；再点一下标签，就能显示出来。
如果不小心关掉了资源管理器，不用慌——通过“窗口”菜单下的“重置窗口布局”就能找回来。

```

## 运行第一个项目

```

创建完项目后，默认打开的是 Program.cs 文件。

Program.cs 是当前项目的主文件，因为它包含了启动项目的结构代码。

为了方便观看，我调整一下显示比例，改为 150%。

注意看：这里有三组花括号，最内层的属于 Main() 方法。

Main 方法很特殊——记住一句话：它是程序执行的唯一入口。

也就是说，咱们写的所有代码，必须放在 Main() 方法的花括号里面，写在外面是没法执行的。

这行代码是模板自带的测试语句，作用是在控制台输出一句 “Hello World”。

什么都不用改，直接点击上方工具栏的“开始执行”按钮（快捷键：Ctrl + F5）。

如果弹出的黑窗口里出现了 “Hello World”，说明代码环境完全正常。

恭喜你！你已经成功运行了第一个 C# 程序。

按任意键可以关闭控制台窗口。

现在你不需要理解每行代码的含义，只需要记住三件事：

Program.cs 是项目的主文件
Main() 方法是程序的入口
运行程序的快捷键是 Ctrl + F5

```
## 创建第二个项目
```

接下来，创建第二个项目

首先，在右侧“解决方案资源管理器“上点右键，找到“添加“

选择“新建项目“

和刚才一样，选择“第一个模板“，点击下一步，把项目名称改为“Lesson1a",继续点击下一步，框架保持默认，最后点击“创建“。

这样，第二个项目就创建好了。

接下来，把第 7 行代码修改为 Hello C#， 修改完之后，先点保存，再点开始运行，养成好习惯。

这时，你会发现：控制台显示的居然还是 “Hello World”，不是 “Hello C#”

这是为什么？

其实很简单：解决方案资源管理器，默认把第一个项目“Lesson1“设为了 “启动项目”。

大家可以对比一下：两个项目名称有什么不同？“Lesson1” 是粗体的，而 “Lesson1a” 不是粗体

粗体代表启动项目。

那怎么运行 Lesson1a 呢？有两种方法：

方法1.在 “Lesson1a” 项目名称上点击右键，找到 “设为启动项”，点击一下就可以；

方法2.点击顶部工具栏的 “启动项目” 下拉框（就在 “开始执行” 按钮旁边），直接切换到 “Lesson1a”。

两种方法都可以，设置好之后，再点击 “开始执行”，这次就能看到控制台输出 “Hello C#” 了，完美！

```
## 解决方案与项目是什么

```

通过刚刚两个项目的创建，大家应该对解决方案和项目有了初步的认识：

右侧的“解决方案资源管理器”，就是专门管理项目的面板。

那到底什么是项目、什么是解决方案呢？

首先说 项目：它不是咱们平时说的 “完整项目”，它只是完整项目的一个组成部分。

一个项目，就是一个负责写代码、编译代码、运行代码的功能单元。

而咱们平时理解的 “完整项目”，其实是 解决方案：它就是一个 “容器”，专门用来管理一个或多个项目。


你可以把解决方案想象成一个书包，项目就是书包里的书，

书包不写字，不干活，只整理书，而书才是用来写代码、实现功能的。

一个书包里可以装多本书，比如 Lesson1、Lesson2 等等。

接下来，咱们关掉 Visual Studio，回到桌面，直观感受一下它们的形式：


从形式上讲，解决方案是一个文件夹，项目也是一个文件夹

一个解决方案文件夹里包含了多个项目文件夹

每个解决方案文件夹下都有一个.slnx 后缀的解决方案配置文件，记录解决方案信息。

每个项目文件夹下都有一个 .csproj后缀的项目配置文件，记录项目信息。

解决方案与项目就是这样协同工作的。

下次你想继续写代码，双击它可以打开整个解决方案。

```
