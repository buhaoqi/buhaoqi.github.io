第一式：站在巨人肩膀上写代码

在写代码之前，一定要明白一个道理：

写代码，从来不是从零开始写。

一开始，我们就是站在巨人肩膀上写代码。

这个巨人，对 C#来说，就是.NET平台。

.NET 平台为开发者提供了两个至关重要的东西：

1. 一个是编译和运行环境
2. 另一个是系统工具库。

系统工具库本质上就是一个命名空间，名字叫System，该空间下定义了大量丰富实用的工具。

它像一个超级军火库，你不需要自己造武器弹药。需要什么直接来库里取就可以了。

比如：这里的 Console，就是从系统工具库中获取的。

这说明什么？

这说明Console 不是凭空出现的，刚刚的修改切断了 Console 这个工具和系统工具库的联系。在脱离了这个巨人提供的军火库后，根本无人认得。

第二式：写代码就是写语句
第三式：先跑起来，再搞明白


村长：
第一句：写代码就是写语句。

第二句：

语句是计算机可以执行的命令。

你写一条语句，就等于向计算机发出一道命令，指挥计算机做一件事，或者说做一个动作。

比如：

```csharp
Console.WriteLine(100);
```
这是一条方法调用语句，使用 writeLine 方法，告诉计算机“把整数 100 输出到控制台“。 

再比如：

```csharp
int score = 80;
```

这是一条声明语句，使用 int 关键字声明一个变量score，并把 80 存储在变量中。

"如果成绩大于等于 60 分，就输出“过关“。可以使用if分支语句。

```csharp
if(chineseScore >= 60){
        Console.WriteLine("过关！");
}
```
现在，你不需要关注语法细节。你只需记住：语句就是计算机能够执行的一个动作。写代码就是写语句。



第三句话：在花括号里面，除了语句，你什么都不能写。



村长： 简单说——语句就是指令。

奇奇： 能举个具体的例子吗？

村长： 当然可以。你在这里输入 `cw`——再按一下键盘上的 `Tab` 键—，系统会自动生成一条语句：  
`Console.WriteLine(123);`  在圆括号里输入 123
它的意思是：指挥计算机向控制台——也就是那个黑窗口——写入一个值：123。

奇奇： 等等——你为什么说它是一条语句？

村长： 好！咱们拆开来看：

- `Console` 是系统提供的一个工具，代表“控制台”，也就是那个黑窗口。
- 实心点 `.` 可以理解为“的”，也就是Console 的。
- `WriteLine` 带有一组圆括号，圆括号表示一个动作，我们称它为“方法”，意思就是“写入一行”。

合起来就是：通过 `Console` 这个工具的 `WriteLine` 方法，向控制台输出123。 
这条语句向计算机发出了一个明确的、清晰的命令。所以——它就是一条语句。
另外，你还要注意一个细节——就是语句末尾的分号。   
分号是语句结束的特征，就像写作文的句号一样，告诉计算机：语句结束。

需要注意的是：这个分号一定是英文输入法下的分号。

奇奇： 那……我只写一个 123 可以吗？

村长：绝对不可以的！

因为只写 123，你写的不是语句。

语句首先

语句一定是指挥计算机执行某个动作。

比如把 123 存储在变量中，
把 123 输出到控制台等等

而123 是语句执行时操作的数据，

123 不是语句。

123 是数据，

数据本身不是代码。  

你啥都没让计算机干，计算机看到一个 123，只会一脸懵：“然后呢？”  
123 要干嘛？打印？保存？还是参与计算？你没说清楚。 


村长： 所以你一定要记住一句话：语句，一定是一个“动作”，而不是一个“东西”。  
数据必须放在语句里才有意义。  
你要是敢单独写一个 123，编译器就敢报错给你看。它根本不知道你想干嘛。

奇奇： 明白了！数据必须放在语句里用。那……写程序就是写语句吗？

村长： 完全正确！  

语句是构成程序的基本单位。  
程序——就是由一条又一条的语句组成的。  
写语句，就是在下命令！

请看这个示例
如果我希望让计算机存储一个考试分数 80
我可以使用变量声明语句

```csharp
int score = 80;
```
如果我希望把这个分数输出到控制台，
可以使用方法调用语句
```csharp
Console.WriteLine(score);
```
如果分数大于等于 60 分时，输出消息：及格
可以使用if 分支语句
```csharp
if(score >= 60)
{
      Console.WriteLine("及格");
}
```
在这里为大家展示了三种常见的语句，不要求大家看懂，只是为了展示一个规则：程序就是由一条一条的语句组成的。

需要注意的是：通常语句都使用分号结尾，但if分支语句后面的花括号也表示语句的结束。所以这里不需要分号结尾。


| 规则一：程序的本质 | 程序是语句的集合。 |
| 规则一：语句的本质 | 语句就是指令，代表一个“动作”。 |
| 规则二：分号的特征 | 分号 `;` 是语句结束的标志，每条语句结尾都必须有分号。 |
| 规则三：数据的角色 | 数据本身不是代码，必须放在语句中使用才有意义。 |

分清代码与数据

代码是 “做事的指令”，是告诉计算机怎么执行、怎么运算、怎么处理的规则。
是你写的 if/else、for 循环、方法、类、逻辑
是静态的指令集合，运行时不能随便修改
作用：控制流程、执行操作
C# 代码示例（纯指令 / 程序）：

数据

数据是 “被处理的内容”，是程序运行时读取、修改、计算的对象。
是变量、数字、字符串、数组、对象的属性值
是动态的内容，运行时可以随意增删改查
作用：被操作、被存储、被传递

✅ 代码 = 规则 / 指令（怎么干）✅ 数据 = 材料 / 内容（干什么）
程序（代码）用来处理数据，数据被程序处理，二者各司其职，绝不混淆。

3. 为什么说 “程序不是数据，数据不是代码”？

① 存储位置不同（C# 底层机制）

代码（程序）：存放在代码段，运行时只读，不能被修改
数据：存放在栈、堆、静态存储区，运行时可读写
C# 会严格隔离这两部分内存，不允许把数据当成指令执行，也不允许把代码当成普通数据修改。
