### ✅ **1. 理解整数类型的本质**
- C# 提供了多种整数类型，主要是为了**节省内存**和**适应不同取值范围的需求**。
- **带符号（signed）**：可以表示正数、负数和零。
- **无符号（unsigned）**：只能表示正数和零（范围更大）。

---

### ✅ **2. 每种类型的实例说明**

#### **① `sbyte` 与 `byte`**
```csharp
sbyte a = -128;      // 最小值
sbyte b = 127;       // 最大值
// sbyte c = 200;    // 错误，超出范围

byte d = 0;          // 最小值
byte e = 255;        // 最大值
// byte f = -5;      // 错误，不能为负
```

#### **② `short` 与 `ushort`**
```csharp
short g = -32768;    // 最小值
short h = 32767;     // 最大值

ushort i = 0;        // 最小值
ushort j = 65535;    // 最大值
```

#### **③ `int` 与 `uint`**
```csharp
int k = -2147483648;   // 最小值
int l = 2147483647;    // 最大值

uint m = 0;             // 最小值
uint n = 4294967295;   // 最大值
```

#### **④ `long` 与 `ulong`**
```csharp
long o = -9223372036854775808L;   // 注意要加 L 后缀
long p = 9223372036854775807L;

ulong q = 0UL;                     // 可以加 UL 后缀
ulong r = 18446744073709551615UL;
```

---

### ✅ **3. 如何选择合适的类型？**
- 如果确定数值不会为负，优先用**无符号类型**，可以扩大可用范围。
- 处理小范围数值时（如年龄、成绩），用 `short` 或 `byte` 节省内存。
- 一般情况用 `int`，它是 C# 中最常用的整数类型。
- 处理大数据、时间戳、大金额时用 `long`。

---

### ✅ **4. 常见错误示例**
```csharp
// 错误：超出范围
byte age = 300;           // 编译错误
int bigNumber = 3000000000; // 超出 int 范围，应使用 long

// 错误：类型不匹配
ushort score = -10;       // 编译错误，不能为负
```

### ✅ **5. 实践建议**
1. **动手写代码**：为每个类型赋最小值、最大值，感受范围。
2. **用 `sizeof()` 查看内存大小**：
   ```csharp
   Console.WriteLine(sizeof(sbyte));  // 输出 1（字节）
   Console.WriteLine(sizeof(int));    // 输出 4
   ```
3. **使用 `MinValue` 和 `MaxValue` 属性**：
   ```csharp
   Console.WriteLine($"int 的最小值：{int.MinValue}");
   Console.WriteLine($"int 的最大值：{int.MaxValue}");
   ```

---

### ✅ **6. 总结**
- 整数类型的选择取决于**取值范围**和**内存占用**。
- 带符号类型支持负数，无符号类型支持更大正数。
- 实际开发中，常用 `int` 和 `long`，特殊情况才会用 `short` 或 `byte`。
