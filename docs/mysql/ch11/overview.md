考点

1. 了解数据库的不安全因素，掌握数据库的安全性的概念；
2. 了解计算机以及信息安全技术方面的系列安全标准；
3. 熟悉数据库的用户管理；
4. 掌握权限的授予、查看和回收；
5. 掌握角色的创建、授权和回收。

## 数据库安全性的含义
数据库安全性指：数据安全和系统安全。

数据安全：不能被非法访问、篡改、破坏、泄露
1. 只允许合法用户在其权限范围内对数据库进行操作


## 本章内容

1. 数据库安全概述
2. 用户管理
3. 权限管理
4. 角色管理


数据库安全性的核心环节可归纳为 **6个核心模块**，它们构成了“从访问控制到数据防护、从审计追溯到应急响应”的全流程安全体系，每个环节层层递进、相互支撑，以下是具体解析（结合教学场景，突出逻辑关联和实用要点）：


### 一、核心环节梳理（含作用+关键技术/操作）
| 核心环节               | 核心作用                                                                 | 关键技术/操作（结合MySQL示例）                                                                 |
|------------------------|--------------------------------------------------------------------------|------------------------------------------------------------------------------------------------|
| 1. 身份认证与访问控制   | 验证用户合法性，限制“谁能登录数据库”                                     | - 用户名+主机+密码认证（MySQL用户标识：`'user'@'主机'`）- 密码策略（复杂度、过期时间）- 双因素认证（高安全场景）- 账户锁定/解锁（`ALTER USER ... ACCOUNT LOCK`） |
| 2. 权限管理             | 控制合法用户“能做什么”，避免越权操作                                     | - 权限分级授权（全局/数据库/表/列级权限）- `GRANT`授权/`REVOKE`回收- 最小权限原则（如仅给查询权限不给删除权限） |
| 3. 数据加密             | 保护数据本身（静态存储+传输过程），防止数据泄露                           | - 传输加密（SSL/TLS，MySQL配置`--ssl-mode=REQUIRED`）- 存储加密（敏感字段加密，如`AES_ENCRYPT`函数）- 数据库文件加密（MySQL Enterprise Edition功能） |
| 4. 数据完整性保护       | 防止数据被非法篡改、破坏，确保数据准确一致                               | - 约束机制（主键、外键、唯一约束、CHECK约束）- 数据校验（如MD5校验）- 事务ACID特性（避免并发修改异常） |
| 5. 审计与日志监控       | 追踪所有数据库操作，便于安全审计和故障追溯                               | - 操作日志（MySQL二进制日志binlog、慢查询日志）- 审计日志（记录用户登录、敏感操作）- 实时监控（如监控异常登录、大量数据删除） |
| 6. 漏洞防护与应急响应   | 抵御外部攻击，处理安全事件（如数据泄露、黑客入侵）                         | - 定期补丁更新（修复数据库漏洞）- 防火墙配置（限制数据库端口访问）- 数据备份与恢复（`mysqldump`备份、Point-in-Time恢复）- 安全事件应急预案（如泄露后数据隔离） |


### 二、各环节的逻辑关联（教学重点）
1. **层层递进关系**：  
   身份认证（第一道门：验证身份）→ 权限管理（第二道门：限制操作）→ 数据加密/完整性（第三道门：保护数据本身）→ 审计监控（事后追溯）→ 漏洞防护（抵御外部攻击）。  
   例：学生登录MySQL后，先通过身份认证（用户名+密码），再通过权限管理仅能查询`student`表，数据加密保护学生手机号，审计日志记录查询操作，漏洞防护避免SQL注入攻击。

2. **核心目标一致**：  
   所有环节最终都是为了实现“数据机密性、完整性、可用性”（安全三要素）：  
   - 机密性：依赖“身份认证+权限管理+数据加密”；  
   - 完整性：依赖“数据完整性保护+审计监控”；  
   - 可用性：依赖“漏洞防护+应急响应”。


### 三、教学应用建议（针对学生理解）
1. 用“银行安全”类比：  
   - 身份认证=银行门禁（刷身份证+密码）；  
   - 权限管理=银行柜台权限（普通柜员不能转账，行长可审批）；  
   - 数据加密=存款信息加密存储；  
   - 审计监控=银行监控摄像头（记录所有操作）。

2. 结合试题场景强化：  
   之前试题中“将学生表查询权限授予用户U1”（对应“权限管理”）、“创建表时设置主键约束”（对应“数据完整性保护”），都是核心环节的实际应用，可让学生关联知识点。


### 总结
数据库安全性的6个核心环节是一个完整的防护体系，缺一不可：  
- 前3个环节（身份认证、权限管理、数据加密/完整性）侧重“主动防护”，阻止安全风险；  
- 后3个环节（审计监控、漏洞防护、应急响应）侧重“被动追溯+风险处置”，应对已发生的安全问题。  

在教学中，可先让学生掌握“身份认证+权限管理+数据完整性”这3个基础环节（对应MySQL实操高频考点），再逐步深入加密、审计等进阶内容。
